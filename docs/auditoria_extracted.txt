SEÇÃO B: PORTFÓLIO DE PROJETOS (O Problema, A Solução e O Valor)
Aqui estão mapeados e auditados todos os 8 projetos reais do seu escopo, estruturados
para demonstrar impacto de negócio e inteligência arquitetural.
1. Samurai Pro (SaaS B2B / Automação e RBI)( EM desenvolvimento )
● O Problema Real: Bloqueios constantes de sistemas anti-bot modernos (WAFs
como Cloudflare/Akamai), rastreamento de fingerprinting e o alto consumo de RAM
que inviabiliza automações de larga escala no acesso a plataformas como o
ChatGPT.
● A Solução Técnica: Construção de uma ferramenta em Python (com UI em tkinter e
execução thread-safe) integrando o undetected-chromedriver. Na nuvem,
implementou arquitetura de Remote Browser Isolation (RBI) utilizando Kernel
Samepage Merging (KSM) no Linux para desduplicação de RAM (reduzindo
consumo de 900MB para <60MB). Integração de IA (GPT-4o Vision) para auto-cura
(Auto-healing) visual.
● Stack Real: Python 3, FastAPI, Selenium, Docker, Linux (KSM).
● Tradução de Valor: Ferramenta de automação desktop e nuvem para evasão de
sistemas anti-bot, demonstrando domínio avançado em infraestrutura de baixo nível,
web scraping e manipulação de DOM.
2. WSP Finance (SaaS B2B2C / Hub de Auditoria) ( EM desenvolvimento )
● O Problema Real: "Bitributação", caos na gestão financeira de PJs/PFs, auditorias
manuais lentas e o alto risco de "alucinações" ao usar IAs comuns para ler dados
fiscais críticos.
● A Solução Técnica: Arquitetura de "IA Híbrida" (Linter Fiscal), isolando a análise
semântica da API do Google Vertex AI do motor matemático determinístico do
Node.js. Implementação de Split Automático de Pagamentos (Pagar.me), isolamento
de tenants via Row-Level Security (RLS) no PostgreSQL e redução do OPEX a zero
nas taxas de egresso via Cloudflare R2 (Cofre Digital criptografado).
● Stack Real: Node.js (Express), TypeScript, Prisma ORM, PostgreSQL, Cloudflare
R2, Google Vertex AI.
● Tradução de Valor: Plataforma financeira desenhada para auditoria automatizada e
otimização extrema de custos na nuvem (FinOps), garantindo consistência
transacional e compliance LGPD.
3. Ecossistema VIVA (HealthTech) ( EM desenvolvimento )
● O Problema Real: O isolamento social de idosos exigindo telemetria clínica sem
fricção (AgeTech), associado à necessidade de extrema privacidade médica (LGPD)
e estabilidade absoluta da API.
● A Solução Técnica: Implementação do padrão Transactional Outbox com filas
BullMQ e Redis no backend NestJS para processamento assíncrono blindado contra
timeouts. Na ponta, orquestração de um RAG Local on-premise via sqlite-vec
(rodando com <30MB de RAM no dispositivo) para garantir privacidade de dados e
modo offline.
● Stack Real: NestJS, PostgreSQL, React/Flutter, Redis, BullMQ, sqlite-vec.
● Tradução de Valor: Ecossistema de saúde focado em Edge AI e processamento
assíncrono de alto volume, garantindo telemetria segura, privacidade total
(Zero-Trust) e alta disponibilidade.
4. Define Pilates SaaS (Sistema de Gestão B2B Multi-tenant)
● O Problema Real: Necessidade de gerenciar múltiplos estúdios (alunos, financeiro,
agendamentos) de forma escalável e unificar o padrão de código de um time de
desenvolvedores em formação.
● A Solução Técnica: Arquitetura Multi-tenant usando o tipo JSONB no PostgreSQL
(para isolamento dinâmico sem reescrever schemas). Liderança técnica (Tech Lead)
focada na implementação de TDD (Test-Driven Development), testes rigorosos e
Code Review como mentoria.
● Stack Real: Django, Django REST Framework (Python), React, Vite,
MySQL/PostgreSQL.
● Tradução de Valor: Sistema de gestão atestando habilidades de liderança técnica,
orquestração de times ágeis e capacidade de entrega Full Stack com cultura de
qualidade de código.
5. Plataforma IBDN (Gestão de Certificação Ambiental)
● O Problema Real: Processos descentralizados, lentos e manuais para o
acompanhamento de plantios ecológicos e emissão de certificados ambientais
corporativos.
● A Solução Técnica: Desenvolvimento de uma API RESTful do zero, protegida por
Role-Based Access Control (RBAC) e JWT. Front-end dinâmico e seguro com
gerenciamento de estados globais.
● Stack Real: FastAPI (Python), MySQL, React, Vite, Zustand, Tailwind CSS.
● Tradução de Valor: Solução web completa com integração Frontend e Backend,
focada em segurança, agilidade e escalabilidade de processos operacionais do
terceiro setor.
6. Pesquisa CNPq (AgroTech & IA)
● O Problema Real: A morosidade e ineficiência das análises químicas de laboratório
tradicionais para classificar os materiais e a rugosidade de amostras de solo.
● A Solução Técnica: Utilização da Física Óptica (Interferometria Speckle) tratada por
algoritmos de busca inteligente (Greedy Best-First Search) construídos em Python
para classificar as amostras em tempo real contra um banco de dados relacional.
● Stack Real / Conceitos: Python, Lógica de Programação Algorítmica, Modelagem
de Bancos de Dados, Análise de Dados Ópticos.
● Tradução de Valor: Pesquisa científica que comprova forte embasamento
matemático e lógico, essenciais para arquiteturas avançadas de Inteligência Artificial
e dados.
7. Site Institucional IBDN (SPA)
● O Problema Real: O instituto precisava de uma vitrine digital corporativa focada em
performance rápida, responsividade plena e otimização para SEO.
● A Solução Técnica: Construção de uma Single Page Application (SPA) pura, com
um motor de roteamento customizado nativo para garantir navegação instantânea
sem sobrecarga de recarregamento.
● Stack Real: HTML5, CSS3, JavaScript ES6+.
● Tradução de Valor: Domínio absoluto dos fundamentos nativos do Front-end e da
construção ágil de interfaces acessíveis.
